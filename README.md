# これは何？

静的サイト用のビルドツールです。
モダンなツールはどれも痒いところに手が届かないため自分好みの仕様で作りました。
小・中規模サイトの構築を想定しています。
BrowserSync によるホットリロード、EJS や CSS、画像ファイルを監視してビルドや圧縮を行います。

更新されていないモジュールが増えたり、Web を取り巻く環境の変化も大きくなったことから、
久々に手を加えて、クラシカルだけどちょっとだけモダンな感じにしました。

## ファイル監視

chokidar を廃止して parcel/watcher を採用しました。

## EJS

なんやかんやで EJS が使い慣れているので採用しています。
`data.json ` で各ページ情報の管理を行います。
EJS に渡している `slug` `base` 等の変数を利用してパンくずリスト等共通パーツの生成を可能にします。
EJS 向けの実用的なソースコードも別途公開したいです。

## CSS

Sass を廃止してモダンブラウザ標準のネストや `@import` による管理を想定していますが、`@import` はビルド時にすべてマージします。
まだこの運用を始めたばかりなので、課題が生じたら本ツールにも手を加えます。

## TypeScript

仮機能として運用中です。
TSC を使用してコンパイルする都合上、一括実行からは実行されず別コマンドで使用する想定です。

## 画像ファイル

JPG 形式と PNG 形式の圧縮、WEBP 形式と AVIF 形式への変換を並列で行います。
AVIF は導入したばかりなので、処理が重ければコマンドを分ける予定です。

## BrowserSync

主にホットリロードの用途で使用します。
HTTPS 化の他、ニッチなところで SSI や Proxy 使用時等のパスの置換にも対応しています。
しかし、肝心の Proxy の設定を入れ忘れていますので、使いたい人は自分で入れてください。

# 使用するには

パッケージ化してインストールしてください。

## Node.js バージョン

`v24.16.0`

# 設定ファイル

下記ファイルで入出力の設定など行います。

```bash
/lib/js/.config.js
```

# コマンド

## 一括実行

```bash
npm run dev
```

## EJS 監視/ビルド

```bash
npm run ejs
```

## CSS 監視/ビルド

```bash
npm run css
```

## 画像ファイル 監視/圧縮/変換

```bash
npm run img
```

## TypeScript 監視/ビルド

```bash
npm run tsc:lib
```

### 本ビルドツール用 TypeScript の監視/ビルド

```bash
npm run tsc:lib
```
